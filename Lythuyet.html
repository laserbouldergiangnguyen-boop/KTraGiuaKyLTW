Phần 1: Trả lời Lý thuyết

1. Sự khác biệt giữa position: absolute và position: relative

Khác biệt: relative giữ nguyên vị trí phần tử trong luồng tài liệu. absolute bứt phá khỏi luồng và định vị theo cha gần nhất có vị trí tương đối.

Ví dụ Responsive: Thường dùng Relative cho khung bao ngoài (card) và Absolute cho nhãn giảm giá ở góc để luôn bám sát khung khi co giãn màn hình.

Deploy lên hosting: Sử dụng các đơn vị linh hoạt như %, vw thay cho px cố định để layout không bị vỡ trên các màn hình thiết bị di động khác nhau.

2. Quy trình đăng ký Domain, Hosting và vai trò DNS

Quy trình: Đăng ký tên miền -> Thuê không gian máy chủ chứa code -> Trỏ bản ghi tên miền về IP máy chủ.

DNS: Chuyển tên miền thân thiện thành IP để trình duyệt kết nối.

Công cụ không cần server: GitHub Pages và Netlify hỗ trợ deploy trực tiếp từ mã nguồn tĩnh.

Bảo mật: Thiếu cấu hình HTTPS hoặc để lộ thông tin cấu hình có thể dẫn đến việc rò rỉ dữ liệu hoặc tấn công giả mạo.

3. So sánh let, const và var

So sánh: var có phạm vi function-scope và có hoisting gán undefined. let/const có phạm vi block-scope an toàn hơn.

// Lỗi logic thường gặp với var trong vòng lặp
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100); // Ra 3, 3, 3 thay vì 0, 1, 2 (khắc phục bằng let)
}

Debug: F12 mở Chrome DevTools, vào tab Sources, chọn dòng code bấm thêm Breakpoint và theo dõi biến ở cửa sổ Scope.

4. Khái niệm Event Delegation

Giải thích: Tận dụng cơ chế nổi bọt (Event Bubbling) để gán 1 trình xử lý sự kiện duy nhất lên thẻ cha thay vì gán cho từng thẻ con đơn lẻ.

Ứng dụng: Giúp các phần tử li động khi render thêm mới tự động nhận sự kiện mà không cần liên kết lại, giảm dung lượng bộ nhớ trình duyệt, tăng tốc hiệu năng xử lý HTML/CSS.
